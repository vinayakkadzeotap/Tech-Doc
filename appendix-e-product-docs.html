Unify

Catalogue · Calculated Attributes · ID Strategy

UNIFY & ENRICH

Consolidate all customer data into a Trusted Golden Record. Standardise attribute taxonomy, compute derived metrics, and define identity resolution rules.

  • Catalogue: Central attribute store; custom or Zeotap verticalised taxonomy; auto field-mapping transformers; DIY interface; hours not days onboarding
  • Calculated Attributes: Aggregate event data over time windows (count, sum); CLV, Recency, Frequency; Real-Time CAs (RTCA); auto-refresh
  • ID Strategy: Patented identity stitching; primary/secondary identifiers; multi-step merge control; unknown-to-known; ID Metadata attributes; UCID
  • Non-Customer Entity Data: Product, location, and other non-person entity profiles

Profile API

REST API · Profiles · Consent · Segments

API

Query unified customer profiles, consent purposes, marketing preferences, and segment membership via REST API. Enables real-time personalisation in any downstream system.

  • POST /cdp/v1/users/_search — Query profile by any identifier
  • POST /cdp/v1/users — Write profile attributes
  • POST /cdp/v1/users/_delete — GDPR profile deletion
  • GET /segments/v1/{id}/members — Segment membership
  • Query by: customer ID, email SHA-256, phone SHA-256, MAID, cookie ID
Consent QueriesSegment Membership GDPR DeleteReal-time Personalisation

S2S HTTP API (Collect)

Server-to-Server data ingestion endpoint

INGEST

Push user, event, page, device, and consent data directly into Zeotap CDP from any server-side system using the write key.

  • Authorisation via write_key per source
  • Payload types: user traits, event (track), page view, device info, consent signals
  • Each call requires at least one identifier (preferably first-party user ID)
