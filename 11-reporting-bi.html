Overview

Zeotap's reporting infrastructure serves two audiences: internal teams (revenue, operations, engineering) and customers (audience insights, activation performance). It is built on a layered data architecture with BigQuery as the primary analytical warehouse, Redshift for customer-facing analytics, and Sisense for embedded dashboards in the Unity product.

Reporting Pipelines

daap-segment-library-reporting

A Spark-based pipeline that reads audience segment metadata from the DAAP library and generates activation performance reports — reach, frequency, match rates — and loads them into Redshift.

-- Segment Performance Report (Redshift)
SELECT
    s.segment_id,
    s.segment_name,
    s.org_id,
    COUNT(DISTINCT sp.ucid)                    AS reached_users,
    COUNT(DISTINCT d.destination_id)           AS active_destinations,
    AVG(act.match_rate)                        AS avg_match_rate,
    SUM(act.impressions)                       AS total_impressions,
    DATE_TRUNC('day', act.activated_at)        AS activation_date
FROM segment_profiles sp
JOIN segments s         ON sp.segment_id = s.id
JOIN activations act    ON sp.segment_id = act.segment_id
JOIN destinations d     ON act.destination_id = d.id
WHERE act.activated_at >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY 1,2,3,8
ORDER BY reached_users DESC;

redshift-reporting-cron

A cron-scheduled service that runs SQL refresh jobs on Redshift, maintaining pre-aggregated views used by Sisense dashboards. Configured with:

{
  "jobs": [
    {
      "name": "audience_size_daily",
      "schedule": "0 3 * * *",
      "query": "CALL sp_refresh_audience_size_daily()",
      "timeout_minutes": 30
    },
    {
      "name": "activation_performance_weekly",
      "schedule": "0 4 * * 1",
      "query": "CALL sp_refresh_activation_performance()",
      "timeout_minutes": 60
    },
    {
      "name": "identity_match_rates",
      "schedule": "0 2 * * *",
      "query": "CALL sp_refresh_identity_metrics()",
      "timeout_minutes": 45
    }
  ]
}

Sisense Embed Token

// Unity Angular — Sisense embed service
@Injectable({ providedIn: 'root' })
export class SisenseService {
  embedDashboard(orgId: number, dashboardId: string, container: HTMLElement): void {
    const jwt = this.authService.getSisenseToken(orgId); // scoped JWT
    SisenseJS.connect({
      server: 'https://analytics.zeotap.com',
      token: jwt,
    }).then(app => {
      app.dashboards.load(dashboardId).then(dashboard => {
        dashboard.attach(container);
        dashboard.refresh();
      });
    });
  }
}

Revenue Reporting

The revenue-reporting-ingestion pipeline automates revenue recognition by ingesting activation events (audience pushes, API calls) and attributing them to commercial agreements:

sequenceDiagram participant Act as Activation Events
(Pub/Sub) participant Ingest as revenue-reporting-ingestion
(Dataflow) participant BQ2 as BigQuery
(revenue_events) participant Enrich as Enrichment Job
(contract lookup) participant Report as Revenue Report
(monthly PDF/CSV) Act->>Ingest: Streaming audience export events Ingest->>BQ2: Write event + metadata BQ2->>Enrich: Join with contract table Enrich->>BQ2: Write enriched revenue rows BQ2->>Report: Scheduled Looker report generation Report-->>Finance: Email delivery 
-- Monthly Revenue Summary (BigQuery)
SELECT
  org_id,
  org_name,
  contract_type,
  SUM(activation_count)    AS total_activations,
  SUM(billable_records)    AS billable_records,
  SUM(revenue_usd)         AS revenue_usd,
  DATE_TRUNC(activated_at, MONTH) AS month
FROM `zeotap-prod.revenue.enriched_activations`
WHERE DATE_TRUNC(activated_at, MONTH) = DATE_TRUNC(CURRENT_DATE(), MONTH) - INTERVAL 1 MONTH
GROUP BY 1,2,3,7
ORDER BY revenue_usd DESC;
