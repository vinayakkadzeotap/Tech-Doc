Platform Architecture Overview

Zeotap CDP is an enterprise Customer Data Platform that unifies identity, enriches profiles, and activates audiences across channels — built on a microservices architecture deployed on Google Cloud Platform and AWS.

What is Zeotap CDP?

Zeotap is a privacy-first Customer Data Platform (CDP) that enables enterprises to collect first-party data, resolve identities across devices and channels, build rich customer audiences, and activate those audiences across 100+ destinations in real time.

The platform sits at the intersection of data engineering, identity resolution, machine learning, and real-time activation, serving telcos, retailers, financial services, and media companies globally.

📡
Collect

Ingest first-party data from web, mobile, CRM, offline, and third-party sources via SDKs and batch pipelines.

🔗
Unify

Stitch identities across devices and channels into a single customer profile using the ID Graph engine.

⚡
Activate

Deliver audiences to DSPs, CRMs, social platforms, and custom destinations in real time or batch.

🤖
Analyze

AI-powered insights, natural language querying via Zoe, segment recommendations, and journey analytics.

System Architecture

Zeotap is built as a collection of domain-oriented microservices organized around the Collect → Unify → Activate data lifecycle. Services communicate via REST, gRPC, and event streaming (Kafka / GCP Pub/Sub).

Event Processing Latency

The full pipeline from event collection to audience membership update is designed for <500ms p99 latency for real-time use cases. Batch ingestion operates on a separate pipeline with configurable scheduling.

Deployment Model

Zeotap runs on a multi-cloud Kubernetes-based infrastructure with GCP as primary and AWS for specific workloads. Services are containerized via Docker and deployed using Helm charts with Harness for progressive delivery.
