Testing Strategy

Zeotap follows a test pyramid with emphasis on fast unit tests at the base, service-level integration tests in the middle, and end-to-end journey regression at the top. All test layers run automatically in CircleCI on every push.

Unit Testing

Scala — Spark Unit Tests (ScalaTest + spark-testing-base)

All Spark-based services (Profile Store, DAAP, identity graph) use ScalaTest with holdenkarau:spark-testing-base for in-process Spark session unit tests:

// ProfileMergeDriverTest.scala
class ProfileMergeDriverTest extends FunSuite with SharedSparkContext {

  test("should merge DPM and Apps data by UCID") {
    val appsData = TestDataGenerator.createAppsDataFrame(spark,
      Seq(
        ("zt_ucid_001", "test@example.com", "2024-10-01"),
        ("zt_ucid_002", "user2@test.com",   "2024-10-02")
      ))

    val dpmData = TestDataGenerator.createDPMDataFrame(spark,
      Seq(
        ("zt_ucid_001", "25-34", "M", "automotive"),
        ("zt_ucid_003", "35-44", "F", "finance")
      ))

    val merged = new MultiDPMergeProcessor()
      .mergeDataframes(appsData, List(dpmData))

    assert(merged.count() == 3)
    assert(merged.filter(col("ucid") === "zt_ucid_001").first()
      .getAs[String]("age_range") == "25-34")
  }

  test("should exclude deleted UCIDs from output") {
    val deletions = spark.createDataset(Seq(DeletionRequest("zt_ucid_del")))
    val profiles  = TestDataGenerator.createProfilesWithDeletion(spark)
    val cleaned   = new DpmIduJoinProcessor().process(deletions, profiles, testOutputPath)
    assert(cleaned.filter(col("ucid") === "zt_ucid_del").count() == 0)
  }
}

Go Unit Tests (segmentestimator)

// bitmap/bitmap_test.go
func TestBitmapAND(t *testing.T) {
    a := NewBitmapFromList([]int{1, 0, 1, 1, 0})
    b := NewBitmapFromList([]int{1, 1, 0, 1, 0})

    result, err := a.AND(b)
    require.NoError(t, err)
    assert.Equal(t, 2, result.Count()) // bits 0 and 3 are both set
}

func TestEstimateSize(t *testing.T) {
    bmp := NewBitmap(1000)
    // Set 100 bits (10% of 1000)
    for i := 0; i < 100; i++ { bmp.Set(i) }

    // With 0.1% sample rate, extrapolated = 100 / 0.001 = 100,000
    estimated := EstimateSize(bmp, 0.001)
    assert.Equal(t, int64(100000), estimated)
}

// cache/querycache_test.go
func TestQueryCacheTTL(t *testing.T) {
    cache := NewQueryCache(2 * time.Second)
    cache.Set("key1", 42)
    assert.Equal(t, 42, cache.Get("key1"))

    time.Sleep(3 * time.Second)
    assert.Nil(t, cache.Get("key1")) // expired
}

Journey Regression Suite

The journey-regression-suite is a Python pytest suite that validates the full end-to-end execution of customer journeys in QA. It sends events, waits for journey execution, then queries BigQuery to verify each step was correctly processed.

Test Structure

journey-regression-suite/
├── config/
│   ├── journeys.yaml        # journey ID mappings (QA workflow IDs)
│   └── env.yaml             # QA environment URLs
├── lib/
│   ├── event_sender.py      # HTTP client for sending test events
│   ├── bq_reader.py         # BigQuery journey trace reader

    ├── test_simple.py        # Basic single-step journey

Test Example — Simple Journey

# tests/test_simple.py
import pytest
from lib.event_sender import send_event
from lib.bq_reader import fetch_journey_traces_by_testuuid_and_workflowId
from lib.verifier import verify_step_execution

@pytest.mark.smoke
@pytest.mark.integration
def test_simple():
    """Validates basic 1-step journey: entry → destination"""

    expected_steps = {
        "08fd76e2-033e-4ce5-972a-396ca6d0eeda",  # entry node
        "6cdbeecc-29ed-4a30-9656-ca97a48ab6f5"   # destination node
    }

    payload = {
        "user": {
            "email_raw": "simple@test.com",
            "vistior_Id": "18894",
            "date_of_birth": "1993-03-15"
        },
        "event": { "eventName": "simpleTest", "countryCode": "US" },
        "country": "US"
    }

    # 1. Send test event to journey engine
    test_uuid = send_event(payload)

    # 2. Poll BigQuery for journey execution trace (max 60s)
    traces = fetch_journey_traces_by_testuuid_and_workflowId(
        test_uuid=test_uuid,
        workflow_id=config["journeys"]["simple_entry"],
        timeout_seconds=60
    )

    # 3. Verify all expected steps were executed successfully
    verify_step_execution(traces, expected_steps, expected_failed=set())

@pytest.mark.regression
def test_entry_condition_filter():
    """Verifies users NOT matching entry condition are excluded"""
    # Send event that should NOT trigger the journey
    payload = {"event": {"eventName": "wrongEvent"}, "country": "DE"}
    test_uuid = send_event(payload)

    traces = fetch_journey_traces_by_testuuid_and_workflowId(
        test_uuid, workflow_id=config["journeys"]["entry_condition"], timeout_seconds=30)

    assert len(traces) == 0, "Journey should NOT have triggered for this event"

@Test
public void testSmartPixelEventIngestion() {
    String payload = readFile("Resources/SmartPixel/smartPixel.json");

    given()
        .header("Authorization", "Bearer " + getApiKey())
        .contentType(ContentType.JSON)
        .body(payload)
    .when()
        .post("/api/v1/events/smartpixel")
    .then()
        .statusCode(200)
        .body("status", equalTo("ACCEPTED"))
        .body("event_id", notNullValue())
        .time(lessThan(500L));  // SLA: < 500ms
}

RBAC Access Control Tests

The backend-automation-framework includes a dedicated RBAC test suite that validates role-based access enforcement across all APIs:

// RBACAPITests.java
public class RBACAPITests {

    @Test
    public void testReadOnlyRoleCannotCreateAudience() {
        String readOnlyToken = getTokenForRole("READ_ONLY", 4521);

        given()
            .header("Authorization", "Bearer " + readOnlyToken)
            .body(createAudiencePayload())
        .when()
            .post("/audience/v1/orgs/4521/audiences")
        .then()
            .statusCode(403);  // Forbidden
    }

    @Test
    public void testMarketerRoleCanCreateAndEditAudience() {
        String marketerToken = getTokenForRole("MARKETER", 4521);
        String audienceId = createAudience(marketerToken);

        given()
            .header("Authorization", "Bearer " + marketerToken)
            .body(editAudiencePayload())
        .when()
            .patch("/audience/v1/orgs/4521/audiences/" + audienceId)
        .then()
            .statusCode(200);
    }

    @Test
    public void testCrossOrgDataIsolation() {
        String org1Token = getTokenForOrg(4521);

        // User from org 4521 should NOT access org 9999 data
        given()
            .header("Authorization", "Bearer " + org1Token)
        .when()
            .get("/audience/v1/orgs/9999/audienceSummaries")
        .then()
            .statusCode(403);
    }
}
