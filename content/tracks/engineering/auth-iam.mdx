**Supported Auth Methods:**
- Email + password (default)
- SSO via SAML 2.0 (enterprise customers)
- SSO via OIDC (Google, Azure AD, Okta)
- MFA via TOTP (Google Authenticator, Authy)

### JWT Token Flow

1. User logs in via Auth0 Universal Login page
2. Auth0 returns `access_token` (JWT, 1-hour TTL) and `refresh_token` (7-day TTL)
3. Frontend stores tokens securely (httpOnly cookies)
4. Every API request includes `Authorization: Bearer {access_token}`
5. API Gateway validates JWT signature, expiration, and claims
6. On token expiry, frontend uses refresh token to get new access token
7. Refresh token rotation prevents token theft replay

**JWT Claims:**
```json
{
  "sub": "auth0|user123",
  "org_id": "org_abc",
  "roles": ["admin", "analyst"],
  "permissions": ["audiences:read", "audiences:write", "destinations:read"],
  "exp": 1705334400
}
```

**Permission Granularity:**
- Resource-level: `audiences`, `destinations`, `profiles`, `reports`, `settings`
- Action-level: `read`, `write`, `delete`, `admin`
- Combined: `audiences:write`, `destinations:delete`, `settings:admin`

### Multi-Tenancy Isolation

Every request is scoped to an organization:

- `org_id` embedded in JWT claims
- API Gateway extracts `org_id` and passes to all downstream services
- Every database query filtered by `org_id`
- No cross-org data access possible at the application layer
- Network-level isolation for enterprise customers (dedicated namespaces)

### API Authentication

For server-to-server integrations:

- **API Keys** — Long-lived keys for SDK and S2S API authentication
- **Service Accounts** — Machine-to-machine OAuth client credentials flow
- **Scoped Tokens** — Limited-permission tokens for specific integrations

### Session Management

- Sessions stored server-side with Auth0
- Inactive sessions expire after 30 minutes (configurable per org)
- Maximum concurrent sessions: 5 per user (configurable)
- Force logout capability for admins (revoke all sessions)

<Callout type="warning" title="Security Best Practices">
Never store JWTs in localStorage — use httpOnly, secure, SameSite cookies. Always validate tokens server-side. Implement token rotation for refresh tokens. Log all authentication events for audit.
</Callout>
