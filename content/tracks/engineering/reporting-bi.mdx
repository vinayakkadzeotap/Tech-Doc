**BigQuery** serves complex analytical queries:
- Full SQL support with JOIN, GROUP BY, window functions
- Columnar storage optimized for aggregation queries
- Partitioned by date and clustered by orgId for cost optimization
- Powers audience insights, overlap analysis, and custom reports

**Apache Druid** serves real-time OLAP queries:
- Sub-second queries on pre-aggregated data
- Real-time ingestion from Kafka (seconds of freshness)
- Powers the Platform Overview dashboard and live event counters
- Rollup aggregation reduces storage while maintaining query speed

### Data Pipeline

**Event to Report Flow:**
1. Events published to Kafka
2. Beam/Dataflow jobs aggregate events into metrics (counts, sums, averages by dimensions)
3. Aggregated metrics written to BigQuery tables and Druid datasources
4. Materialized views pre-compute common report patterns (hourly)
5. Query API reads from the appropriate engine based on query type
6. Dashboard UI renders visualizations

### Query Optimization

- **Partition pruning** — Queries scoped to date range only scan relevant partitions
- **Clustering** — BigQuery tables clustered by orgId for fast filtering
- **Query caching** — Identical queries within 1 hour served from cache
- **Materialized views** — Common aggregations pre-computed on schedule
- **Query quotas** — Per-org query limits prevent runaway costs

<Callout type="info" title="Cost Management">
BigQuery is billed per bytes scanned. Partition pruning and clustering reduce costs by 80-90%. The analytics team monitors per-org query costs and optimizes expensive patterns.
</Callout>
