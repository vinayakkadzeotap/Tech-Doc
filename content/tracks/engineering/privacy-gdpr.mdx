## 🔒 Privacy & GDPR

Privacy is not a feature — it's an architectural principle. Zeotap was born in Berlin with GDPR in its DNA. This module covers how privacy is enforced at every layer of the platform.

### Consent Management

**Consent Collection:**
- Web SDK integrates with CMP (OneTrust, Cookiebot, etc.) via TCF 2.0 API
- Every event tagged with consent state at collection time
- Consent purposes: `analytics`, `marketing`, `personalization`, `advertising`
- GPC (Global Privacy Control) signal respected automatically

**Consent Storage:**
- Consent state stored per UCID in the Profile Store
- Consent history maintained (when consent was given/withdrawn)
- Immutable audit log of all consent changes

**Consent Enforcement:**
- Every pipeline stage checks consent before processing
- Kafka consumer validates consent before writing to Profile Store
- Audience Engine only evaluates consented profiles
- Activation layer only sends data to destinations with matching consent

### Right to Erasure (DSAR)

When a data subject requests deletion:

1. **Request received** via API or admin UI with identifier (email, phone, UCID)
2. **Identifier resolved** to UCID via Identity Graph
3. **Deletion cascade** across all data stores:

  headers={["Store", "Data Deleted", "Method"]}

    ["ML Features", "Feature vectors", "Feature store purge"],

4. **Verification** — Automated check confirms data removal across all stores
5. **Audit log** — Deletion request and completion logged (metadata only, no PII)
6. **SLA** — Complete erasure within 72 hours (GDPR requires 30 days)

- Data plane runs independently in each region — no cross-region data transfer
- Control plane (org config, user management) is shared but contains no PII
- Network isolation via VPC peering — no public internet exposure for data

### Encryption

- **At rest:** AES-256 encryption for all data stores (BigTable, BigQuery, GCS)
- **In transit:** TLS 1.3 for all service-to-service communication
- **Key management:** Google Cloud KMS with customer-managed encryption keys (CMEK) option
- **PII hashing:** All PII identifiers SHA-256 hashed before storage in Identity Graph

### Audit Logging

Every data access and modification is logged:
- Who accessed what data, when, from where
- Structured logs in Cloud Logging
- Retention: 2 years (configurable per regulation)
- Exportable for compliance audits
