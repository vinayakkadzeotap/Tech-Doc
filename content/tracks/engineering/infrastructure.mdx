**Autoscaling:**
- Horizontal Pod Autoscaler (HPA) based on CPU/memory/custom metrics
- Cluster Autoscaler adjusts node count based on pending pods
- Vertical Pod Autoscaler (VPA) recommends resource requests
- Scale-to-zero for non-critical batch workloads during off-peak

### Terraform Modules

The `gcp-terraform-modules` repository contains reusable modules:

```
gcp-terraform-modules/
├── modules/
│   ├── gke-cluster/      # GKE cluster configuration
│   ├── bigtable/          # BigTable instance + tables
│   ├── bigquery/          # Datasets, tables, views
│   ├── kafka/             # Confluent Kafka on GKE
│   ├── redis/             # Memorystore Redis
│   ├── networking/        # VPC, subnets, firewall rules
│   ├── monitoring/        # Prometheus, Grafana stack
│   └── iam/               # Service accounts, roles
├── environments/
│   ├── dev/
│   ├── staging/
│   └── production/
└── global/                # Cross-environment resources
```

**State Management:**
- Terraform state stored in GCS bucket with versioning
- State locking via GCS to prevent concurrent modifications
- Separate state files per environment and module

- Each region runs a complete, independent data plane
- No customer data crosses region boundaries
- Control plane services (auth, org config) are global
- DNS-based geo-routing directs users to nearest region

### Networking

- **VPC per region** — Isolated networks with private subnets
- **Cloud NAT** — Managed egress for GKE pods (no public IPs on pods)
- **Internal Load Balancer** — Service-to-service within cluster
- **External Load Balancer** — Customer-facing APIs with Cloud Armor WAF
- **Cloud CDN** — Static assets and SmartPixel content
- **Private Google Access** — GCP services accessed without public internet

### Secret Management

- **Google Secret Manager** — API keys, database passwords, service credentials
- **External Secrets Operator** — Syncs secrets from Secret Manager to K8s secrets
- **Rotation** — Automated secret rotation for service accounts (90-day cycle)
- **Access control** — IAM-based access to secrets per service account

<Callout type="info" title="Infrastructure Cost">
Infrastructure cost is a significant line item. Every engineer should consider cost implications of their architectural decisions. Prefer managed services (BigTable, BigQuery) over self-hosted alternatives. Use preemptible instances for fault-tolerant workloads.
</Callout>
