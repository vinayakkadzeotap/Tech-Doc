Infrastructure Overview

Zeotap's infrastructure is managed entirely as code across two cloud providers — GCP (primary) and AWS (secondary workloads). All resources are defined in Terraform modules with Terragrunt wrappers for environment promotion (Dev → QA → Production).

GCP Terraform Modules

Terragrunt Configuration (DRY)

# gcp-terraform-modules/terragrunt.hcl
# Root terragrunt config — shared across all environments

locals {
  env    = get_env("ENV", "dev")     # dev | qa | prod
  region = "europe-west1"
  project_map = {
    dev  = "zeotap-dev-123"
    qa   = "zeotap-qa-456"
    prod = "zeotap-prod-789"
  }
  project_id = local.project_map[local.env]
}

inputs = {
  project_id = local.project_id
  region     = local.region
  env        = local.env
  labels = {
    env     = local.env
    team    = "platform"
    managed = "terraform"
  }
}

GKE Cluster Module Usage

# environments/prod/gke-cluster/terragrunt.hcl
terraform {
  source = "git::https://github.com/zeotap/gcp-terraform-modules.git//gke-cluster?ref=v1.2.0"
}

include "root" {
  path = find_in_parent_folders()
}

inputs = {
  cluster_name         = "zeotap-prod-cluster"
  node_count           = 12
  machine_type         = "n2-standard-8"
  disk_size_gb         = 100
  preemptible          = false
  min_node_count       = 6
  max_node_count       = 30
  enable_autoscaling   = true
  network              = "zeotap-vpc-prod"
  subnetwork           = "zeotap-subnet-prod"
  master_ipv4_cidr     = "172.16.0.32/28"
}

Ansible Configuration Management

The Ansible, ansible-scripts, ansible_playbooks, and ansible_roles repositories manage server configuration for non-Kubernetes workloads: Druid cluster, Qubole nodes, jump hosts, and legacy VMs.

Playbook Structure

├── site.yml                    # master playbook
├── druid/
│   ├── install-druid.yml       # Druid cluster setup
│   └── configure-druid.yml     # segment estimator Druid config
├── monitoring/
│   ├── prometheus-node.yml     # node_exporter on all hosts

Networking Architecture

# VPC + NAT + Peering module
module "vpc_nat_peering" {
  source  = "gcp-terraform-modules//vpc-nat-peering"
  version = "v1.0.0"

  vpc_name         = "zeotap-vpc-prod"
  subnet_cidr      = "10.10.0.0/20"
  nat_name         = "zeotap-nat-prod"
  nat_ip_count     = 4              # static egress IPs for partner allowlisting
  peer_network_ids = []             # Cloud SQL, BigTable private service access
  enable_private_google_access = true
}
Private Google Access — All GKE pods communicate with Google services (BigTable, BigQuery, GCS, Pub/Sub) over private Google access, without public internet traversal. This ensures data never leaves Google's backbone network.
