GenAI / Zoe — AI-Powered Analytics

Zoe is Zeotap's embedded AI analyst. It converts natural language questions into BigQuery SQL, translates SQL WHERE clauses into Zeotap Audience JSON, and powers an agentic server with conversational analytics — all backed by Google Gemini and Vertex AI.

Python 3.10+ · FastAPI
Google Gemini · Vertex AI
MCP Protocol

Overview

The Zeotap GenAI layer comprises three interconnected services that bring natural language intelligence to the CDP:

💬
Zoe API (zeotap-genai)

FastAPI service: NL→SQL conversion, SQL validation, SQL→Audience JSON transformation. Org-scoped schema management.

🧠
Agentic Server

Google ADK + FastAPI: Multi-turn conversational analytics agents with memory, tool use, and RBAC authentication.

🔌
MCP Server (zeotap-mcp)

Model Context Protocol server: Schema discovery, feature analysis, and query building — exposed to Claude Desktop and Claude Code.

🔍
Insight Agent

Automated insight generation engine that scans audience data and surfaces anomalies, trends, and recommendations.

Zoe API — Endpoint Reference

Base URL: https://zoe-api.zeotap.com (prod) / http://0.0.0.0:5000 (local)
Interactive docs: /docs (Swagger UI)

POST /zoe/message Natural language → SQL

Converts a natural language prompt into a BigQuery SQL query scoped to the organization's schema. Returns the SQL, an explanation, and optional chart metadata.

Request

json
{
  "orgId":  "1234",            // Organization ID (string or number)
  "prompt": "Show me users who made a purchase in the last 30 days with order value above $100, grouped by country"
}

Response

json
{
  "success": true,
  "sql": "SELECT profile.country, COUNT(DISTINCT profile.ucid) AS users\nFROM `org_1234.events` e\nJOIN `org_1234.profile` p ON e.ucid = p.ucid\nWHERE e.event_name = 'purchase'\n  AND e.order_value > 100\n  AND e.timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY)\nGROUP BY profile.country\nORDER BY users DESC",
  "explanation": "This query finds unique purchasers in the last 30 days with order value > $100, grouped by their home country.",
  "chart_type": "bar",
  "tokens_used": 847
}
POST /zoe/sql_to_audience SQL WHERE clause → Audience JSON

Parses a BigQuery SQL WHERE clause using sqlglot and converts it into Zeotap's Audience JSON format for use in the Audience Builder UI.

NL → SQL → Audience: End-to-End Example

💬 Natural Language Input
"Male users aged 25-40 who purchased electronics in the last 60 days"
🗄️ SQL WHERE Clause (intermediate)
profile.gender = 'male'
AND profile.age BETWEEN 25 AND 40
AND event.category = 'electronics'
AND event.timestamp > CURRENT_TIMESTAMP - INTERVAL 60 DAY

Request

json
{
  "sql": "SELECT * FROM users WHERE profile.gender = 'male' AND profile.age BETWEEN 25 AND 40 AND event.category = 'electronics' AND event.timestamp > CURRENT_TIMESTAMP - INTERVAL 60 DAY",
  "orgId": "1234"
}

Response — Zeotap Audience JSON

json
{
  "success": true,
  "audienceJson": {
    "operator": "AND",
    "rules": [
      {
        "ruleType": "user_trait_rules",
        "operator": "AND",
        "conditions": [
          { "field": "gender", "op": "eq", "value": "male" },
          { "field": "age", "op": "between", "value": [25, 40] }
        ]
      },
      {
        "ruleType": "event_rules",
        "operator": "AND",
        "lookbackWindow": { "unit": "DAY", "value": 60 },
        "conditions": [
          { "field": "category", "op": "eq", "value": "electronics" }
        ]
      }
    ]
  },
  "tablesUsed": ["profile", "event"]
}

Agentic Server (zeotap-agentic-server)

The agentic server uses Google's Agent Development Kit (ADK) to orchestrate multi-turn conversations with data context. It maintains session state, executes tools (SQL queries, schema lookups), and streams responses.

🤖 Agentic Server — Request Lifecycle
sequenceDiagram participant User participant FastAPI as FastAPI Server participant Auth as RBAC Auth participant ADK as Google ADK Agent participant Tools as Tool Registry participant BQ as BigQuery User->>FastAPI: POST /agent/chat
{session_id, message, org_id} FastAPI->>Auth: Validate RBAC token Auth-->>FastAPI: User context + permissions FastAPI->>ADK: RunAgent(context, message) ADK->>ADK: Select reasoning strategy loop Tool Calls (ReAct loop) ADK->>Tools: Call tool(name, args) alt SQL Execution Tools->>BQ: Execute query BQ-->>Tools: Results else Schema Lookup Tools->>Tools: Cached schema end Tools-->>ADK: Tool result end ADK-->>FastAPI: Final response + citations FastAPI-->>User: {response, sql_used, data_preview}

MCP Server (zeotap-mcp)

The MCP (Model Context Protocol) server exposes Zeotap's data capabilities directly to AI assistants like Claude. It provides three core tools: schema discovery, feature analysis, and query building.

🔌
Claude Desktop Integration

The MCP server can be connected to Claude Desktop or Claude Code to enable natural language access to Zeotap data from within your IDE or chat interface. Requires VPN connectivity to internal Zeotap infrastructure.

Setup & Configuration

1
GCP Authentication

Authenticate with Google Cloud using gcloud auth login. Ensure the service account has BigQuery Data Viewer and BigTable Reader roles.

2
Environment Variables

Create zoe/.env with GOOGLE_PROJECT_ID, GOOGLE_LOCATION, and ZEOTAP_ACCESS_TOKEN.

3
Start the Server

Run python app.py from the zoe/ directory. API docs available at http://0.0.0.0:5000/docs.

4
Register Organization

Call POST /zoe/add_new with your org_id, org_name, and auth token to register the org and load its schema.

VariableRequiredDefaultDescription
GOOGLE_PROJECT_ID Required — GCP project for BigQuery queries
GOOGLE_LOCATION Required — GCP region (e.g. europe-west1)
ZEOTAP_ACCESS_TOKEN Required — Bearer token for Zeotap API calls
USE_MOCK_API Optional false Use mock data (no GCP required)
REDIS_HOST Optional localhost Redis host for schema caching
BIGQUERY_PROJECT Optional zeotap-cdp-prod-datascience BigQuery project (MCP server)
