Glossary of Terms

Definitions for technical terms, acronyms, and Zeotap-specific concepts used throughout this documentation. Terms are alphabetically ordered with cross-references to relevant chapters.

Navigation Tip — Click any letter in the sidebar to jump to that alphabetical section. Cross-reference links in format [Ch N] point to the corresponding chapter.

A

Activation
The process of pushing audience segments and profiles to downstream marketing destinations, ad platforms, or CRM systems. Zeotap's activation layer encompasses audiences, destinations, journeys, and real-time exports. See Ch 8 — Data Activation.
ADK (Agent Development Kit)
Google's framework for building multi-agent AI systems, used in journeys-ai. Provides SequentialAgent, LoopAgent, and tool-calling primitives. See Ch 7 — Customer Journeys.
Aerospike
An in-memory NoSQL key-value store used by Zeotap's Profile Store for low-latency identity lookups. Monitored via a custom Prometheus exporter on port 9999. See Ch 16 — Observability.
AI Attributes
Machine-learning-derived attributes appended to the Unified Customer Profile by Zeotap's ML Platform. Examples: ai_propensity_score, ai_ltv_score, ai_lookalike_score, ai_churn_risk. See Ch 10 — ML Platform.
Airflow
Apache Airflow, an open-source workflow orchestration platform. Deployed in GCP as Cloud Composer (managed Airflow). Used for batch ETL, data partner ingestion, and reporting pipelines. See Ch 3 — Data Ingestion.
Alertmanager
Prometheus component that routes, deduplicates, and silences alerts. Configured with three notification channels: Slack, PagerDuty, and email. Routes by team label and severity: page. See Ch 16 — Observability.
ANN (Approximate Nearest Neighbour)
An algorithm for finding dataset members most similar to a query vector without exhaustively checking all candidates. Used by the Segment Estimator's bitmap engine and Atlas AI's lookalike modelling. See Ch 6 and Ch 10.
Ansible
An agentless configuration management tool using YAML playbooks. Zeotap uses it for Druid cluster setup, Qubole node configuration, jump host hardening (CIS benchmark), and SSL certificate renewal. See Ch 15 — Infrastructure.
Artifact Registry (GAR)
Google Artifact Registry — the container image registry used to store Docker images for all Zeotap microservices. Region: europe-west1. See Ch 17 — CI/CD.
Atlas AI
Zeotap's lookalike modelling engine. Computes cosine similarity between seed audience profile embeddings and the full customer base using Vertex AI Matching Engine for ANN search. See Ch 10 — ML Platform.
Audience
A named, rule-defined segment of customer profiles. Audiences are built via the Audience Builder UI (drag-and-drop conditions), estimated in real time by the Segment Estimator, and activated via Destinations. See Ch 6 — Audience Management.

B

Batch Ingest
The bulk loading of first-party or partner data via file upload (S3/GCS CSV/Parquet) or SFTP, as opposed to streaming ingestion. Handled by Cloud Composer DAGs and Dataflow pipelines. See Ch 3.
BG/NBD Model
Beta-Geometric/Negative Binomial Distribution — a probabilistic model for predicting customer purchase frequency and lifetime in a non-contractual subscription setting. Used in Zeotap's LTV scoring pipeline. See Ch 10.
BigQuery (BQ)
Google's serverless, fully managed data warehouse. Primary analytics store for Zeotap reporting, audience counts, and ML feature tables. Separate projects for prod-datalake and prod-microsvcs environments. See Ch 11 — Reporting & BI.
BigTable
Google's NoSQL wide-column store, used as the primary Identity Graph store mapping hash identifiers (email SHA-256, phone SHA-256, device IDs) to canonical zt_id keys. See Ch 3 — Identity Resolution.
Bitmap
A compact bitset representation used by the Segment Estimator to count unique users matching an audience condition. 64-bit uint64 words with Hamming-weight (popcount) operations for O(n/64) population estimation. See Ch 6.

C

CAPI (Conversions API)
Meta (Facebook) Conversions API — a server-side pixel firing API used by SmartPixel-3P to send conversion events directly from Zeotap's servers, bypassing browser-side ad-blocking. See Ch 8.
CDF (Change Data Feed)
Delta Lake feature that captures row-level changes (INSERT/UPDATE/DELETE) to a Delta table for downstream consumption. Used in Zeotap's Profile Store to feed incremental merge jobs. See Ch 5.
CDP (Customer Data Platform)
A software category that unifies first-party customer data from multiple sources into a single, persistent customer profile. Zeotap's CDP consists of Collect, Unify, and Activate product pillars. See Ch 1 — Platform Overview.
CircleCI
Cloud-based continuous integration platform. Zeotap uses CircleCI for SBT (Scala/Play) and Go builds, with JaCoCo test coverage reporting, Codacy integration, and Google Artifact Registry pushes. See Ch 17 — CI/CD.
Cloud Composer
Google Cloud's managed Apache Airflow service. Used for orchestrating batch ETL, partner data ingestion, and reporting DAGs. Configured via the cloud-composer Terraform module. See Ch 15.
Cloud SQL
Google's fully managed relational database service. Used by Zeotap for Audiences database (PostgreSQL), GDPR consent/erasure records, and various microservice metadata stores. See Ch 15.
Codacy
Automated code quality and coverage reporting tool. Integrated with CircleCI via codacy-coverage-reporter to upload JaCoCo XML coverage reports after each build. See Ch 17.
Consent String
A base64-encoded binary string conforming to IAB TCF v2.2 format, storing a user's vendor consents and purpose permissions as a bitfield. Parsed and stored in Zeotap's GDPR backend. See Ch 13 — Privacy & GDPR.
Cookie ID
A browser-set persistent identifier stored in a first-party (z_uuid) or third-party cookie. Used as a probabilistic bridge between anonymous browsing sessions and known profiles. See Appendix B.
CRO (Conversion Rate Optimisation)
A marketing practice of improving website or campaign experience to increase the percentage of visitors who complete desired actions. Supported by Zeotap's propensity scoring and journey optimisation. See Ch 10.

D

DAAP (Data As A Product)
Zeotap's internal framework treating datasets as first-class products, with defined schemas, SLOs, and ownership. The daap-processing service handles profile enrichment pipelines. See Ch 5.
Data Partner
A third-party data provider whose audience segments or identity data is ingested into the Zeotap platform for enrichment. Managed in the zeotap-prod-datapartner GCP project. See Ch 3.
Dataflow
Google Cloud Dataflow — a fully managed Apache Beam stream/batch processing service. Used for real-time event enrichment, erasure processing, ML batch scoring, and reporting ingestion pipelines. See Ch 3.
Dataproc
Google Cloud Dataproc — managed Apache Spark/Hadoop clusters. Used for large-scale batch profile merging, audience export jobs, and ML feature engineering. Two Terraform modules exist: persistent dataproc and single-job ephemeral-dataproc. See Ch 15.
Delta Lake
An open-source storage layer that adds ACID transactions, schema enforcement, and time travel to Apache Spark data lakes. Used as the foundation of Zeotap's Profile Store, storing Unified Customer Profiles as Delta tables. See Ch 5.
Destination
A configured integration endpoint where Zeotap exports audience data (e.g., Google Ads, Meta, Salesforce, S3 file exports). Managed by the data-destination microservice. See Ch 8 — Data Activation.
DPM (Data Partner Mapping)
The process of mapping third-party data partner identifiers to Zeotap's canonical zt_id via the Identity Graph, enabling cross-partner data enrichment. See Ch 5.
Druid
Apache Druid — a real-time analytics database optimised for sub-second queries on large time-series datasets. Used by Zeotap's Segment Estimator for fast audience size approximation queries. Runs on AWS EC2 (Druid on EC2 Terraform module). See Ch 6.

E

Enrichment
The process of augmenting a customer profile with additional attributes from first-party events, second-party data partner graphs, or AI-derived scores. Processed by the Profile Store merge pipeline. See Ch 5.
Erasure Pipeline
The GDPR-compliant data deletion workflow that fans out a Right-to-Erasure request across all data stores (Profile Store Delta, BigTable, BigQuery, Audiences DB, Destinations). Triggered via Pub/Sub and implemented with Dataflow jobs. See Ch 13.
Event
A discrete, timestamped user action collected by the Zeotap SDK or API (e.g., page_view, add_to_cart, purchase). Events are the primary raw data unit flowing through the Collect pipeline. See Ch 2 — Data Collection.
Extrapolation
In Segment Estimation, the statistical scaling of a sampled audience count (from a subset shard) to the full population size. Used when bitmap shard counts are available for only a fraction of users. See Ch 6.

F

Feature Store
A centralised repository of ML features (pre-computed attributes) used for model training and online scoring. Zeotap's feature store is backed by BigQuery for batch features and Aerospike for low-latency online features. See Ch 10.
Firebase
Google's mobile and web application development platform. Zeotap uses Firebase Authentication as the identity provider, with the Admin SDK validating Firebase ID tokens in the Akbar authenticator. See Ch 14 — Auth & IAM.
First-Party Data
Data collected directly from a brand's own customers and interactions (website, app, CRM, purchase history). Contrasted with second-party (partner-shared) and third-party (purchased) data. Zeotap's Collect and Unify products process first-party data. See Ch 1.

G

GAID (Google Advertising ID)
A resettable advertising identifier for Android devices, formatted as a UUID. Used as a mobile identity bridge in Zeotap's ID Graph. Type value: gaid. See Appendix B.
Gamma-Gamma Model
A probabilistic model that estimates the average transaction value for a customer, conditioned on them being active. Combined with BG/NBD to produce Customer Lifetime Value (LTV) scores. See Ch 10.
GCE Service Discovery
Prometheus's gce_sd_configs mechanism that auto-discovers GCP VM instances by project and zone, using GCE metadata labels (monitoring=yes) to filter scrape targets. See Ch 16.
GDPR (General Data Protection Regulation)
The EU regulation governing the collection, processing, and storage of personal data. Zeotap implements GDPR compliance through consent management, erasure pipelines, and IAB TCF v2.2. See Ch 13 — Privacy & GDPR.
GKE (Google Kubernetes Engine)
Google's managed Kubernetes service. All Zeotap microservices run in GKE clusters organised into five namespaces: collect, unify, activate, ai, and infra. See Ch 15.
Grafana
An open-source observability and data visualisation platform. Zeotap uses Grafana for infrastructure dashboards (Node Exporter, Scylla, Kafka) and application SLO dashboards, backed by Prometheus data sources. See Ch 16.

H

Harness
A modern continuous delivery platform used by Zeotap for production deployments. Implements a multi-stage pipeline: QA deploy → manual approval gate → canary 10% → full production rollout. See Ch 17 — CI/CD.
HEMS (Hashed Email Match String)
A standardised, hashed version of an email address used for privacy-safe identity matching across partners. Typically SHA-256 of lowercase, trimmed email. See Appendix B.
Helm
The Kubernetes package manager. Zeotap uses Helm charts to template K8s manifests for each microservice, with environment-specific values-prod.yaml / values-qa.yaml overrides. See Ch 17.

I

IAB TCF (Transparency and Consent Framework)
An industry standard developed by IAB Europe for managing user consent for advertising data processing. Zeotap implements TCF v2.2 for consent string parsing and vendor permission management. See Ch 13.
ID Graph
A persistent graph data structure mapping multiple device, cookie, and identity identifiers to a single canonical zt_id for a person. Stored in BigTable, built by the id-graph-service. See Ch 3 — Identity Resolution.
IDFA (Identifier for Advertisers)
Apple's resettable advertising identifier for iOS devices. Used as a mobile identity bridge in Zeotap's ID Graph. Type value: idfa. See Appendix B.
Ingestion
The process of loading raw data (events, profiles, partner segments) into Zeotap's data infrastructure. Supports two modes: batch (S3/GCS/SFTP file upload) and streaming (SDK → Pub/Sub → Dataflow). See Ch 3 — Data Ingestion.

J

JaCoCo
Java Code Coverage library. Used in Zeotap's SBT build pipeline (via sbt jacoco) to generate XML coverage reports uploaded to Codacy. See Ch 17.
Jaccard Similarity
A measure of overlap between two sets, computed as |A ∩ B| / |A ∪ B|. Used by the Segment Estimator to calculate the percentage overlap between two audience segments. See Ch 6.
Journey
An automated, multi-step marketing workflow in Zeotap's Symphony Workflow Engine. Consists of five block types: ENTRY_CONDITION, CONDITIONAL_SPLIT, MULTIBRANCH, DELAY, and DESTINATION. See Ch 7 — Customer Journeys.
JFrog Artifactory
A universal binary repository manager. Used by Zeotap alongside Google Artifact Registry to store Maven artifacts (JARs), SBT packages, and shared library releases. See Ch 17.
JOLT
JSON-to-JSON transformation library. Used by the Akbar authenticator to transform Firebase JWT claims into Zeotap-specific JWT payload format using a declarative spec file. See Ch 14.
JWT (JSON Web Token)
A compact, URL-safe token format used for stateless authentication. Zeotap issues JWTs with a 1-hour TTL from the Akbar service, signed with HMAC-SHA256. Frontend SDKs perform silent refresh before expiry. See Ch 14.

K

Kafka
Apache Kafka — a distributed event streaming platform. Used by Zeotap as the primary event bus between Collect SDK ingestion and downstream processing pipelines. Monitored via kafka-monitor-connect exporters in EU, US, and IN regions. See Ch 16.
Kafka Lag
The difference between the latest offset produced on a Kafka topic and the offset last consumed by a consumer group. Zeotap monitors consumer lag as a key SLO indicator. See Ch 16.

L

LightGBM
A gradient boosting framework by Microsoft, optimised for speed on tabular data. Used alongside XGBoost for churn prediction and propensity scoring in Zeotap's ML pipeline. See Ch 10.
Lookalike Modelling
An ML technique that identifies users who share similar attributes to a seed audience (e.g., known purchasers), enabling expansion to high-propensity prospects. Zeotap's implementation uses Vertex AI Matching Engine (Atlas AI). See Ch 10.
LRU Cache
Least-Recently-Used cache — an eviction policy that removes the least recently accessed item when capacity is full. The Segment Estimator uses an LRU cache (TTL 7200s) for audience size query results. See Ch 6.
LTV (Customer Lifetime Value)
A prediction of the total revenue a business can expect from a customer over the duration of their relationship. Zeotap's ML platform computes LTV using BG/NBD + Gamma-Gamma models, producing ai_ltv_score. See Ch 10.

M

MDM Delta Table
The Master Data Management Delta table — Zeotap's canonical persistent store for Unified Customer Profiles, partitioned by zt_id with full ACID transactional semantics. See Ch 5.
Mermaid
A JavaScript-based diagramming and charting tool using a Markdown-inspired syntax. Used throughout this documentation to render architecture diagrams, sequence flows, and pipeline flowcharts. See mermaid.js.org.
Microservice
An independently deployable service with a bounded domain responsibility. Zeotap's platform is entirely microservice-based, running in GKE with each service deployed from its own repository and CI/CD pipeline. See Ch 1.
MultiDPMergeProcessor
A Scala Spark processor in the Profile Store merge pipeline that handles merging attributes from multiple data partners into a single Unified Profile, applying priority rules and conflict resolution logic. See Ch 5.

N

NgRx
A reactive state management library for Angular, inspired by Redux. Zeotap's Unity frontend uses NgRx for predictable, centralized state management with Actions, Reducers, Effects, and Selectors. See Ch 12 — Unity Dashboard.
Node Exporter
A Prometheus exporter that exposes host-level metrics (CPU, memory, disk, network) from Linux servers. Deployed on all GCE instances via Ansible's prometheus-node.yml playbook on port 9100. See Ch 16.

O

OAuth 2.0
An open standard for access delegation. Zeotap's data-destination service manages OAuth tokens for partner integrations (e.g., Google Ads, Meta), handling refresh token rotation and expiry detection automatically. See Ch 8.
Opt-Out
A user's exercise of their right to request that their personal data no longer be processed for marketing purposes. Stored in Zeotap's GDPR backend (OptOutUsers table) and propagated to all downstream systems. See Ch 13.

P

PagerDuty
An incident management and alerting platform. Zeotap routes severity: page alerts from Alertmanager to PagerDuty for on-call engineer notification, with a 30-minute repeat interval. See Ch 16.
Play Framework
A high-performance, reactive web framework for Scala and Java. Used by multiple Zeotap backend services including audiences, data-destination, and gdpr-backend. See Ch 6, Ch 8, Ch 13.
Profile Store
Zeotap's core data persistence layer for Unified Customer Profiles. Built on Delta Lake (Apache Spark + HDFS/GCS), implementing a full merge pipeline for multi-source identity and attribute consolidation. See Ch 5 — Profile Store.
ProfileMergeDriver
The main Scala Spark class (com.zeotap.profile.merge.drivers.ProfileMergeDriver) that orchestrates the Profile Store merge pipeline, reading CDF from Delta Lake and applying merge processors. See Ch 5.
Prometheus
An open-source systems monitoring and alerting toolkit. Zeotap's primary metrics backend, scraping 5 GCP projects across 9 zones (EU, US, APAC) with a 1-minute scrape interval. See Ch 16 — Observability.
Propensity Score
A probability (0–1) predicting the likelihood a customer will perform a specific action (purchase, churn, upgrade). Computed by Zeotap's ML platform using XGBoost/LightGBM and stored as ai_propensity_score. See Ch 10.
Pub/Sub
Google Cloud Pub/Sub — a fully managed asynchronous messaging service. Used as Zeotap's event bus between Collect ingestion, Dataflow processing, identity resolution, and erasure pipelines. See Ch 3.

Q

Qubole
A cloud-based big data platform that provides managed Apache Spark, Hive, and Presto clusters. Used by Zeotap for some historical analytics workloads. Configured by Ansible playbooks. See Ch 15.

R

RBAC (Role-Based Access Control)
An access control model where permissions are assigned to roles, and roles are assigned to users. Zeotap's Akbar authenticator enforces RBAC, with roles like ADMIN, OPERATOR, and VIEWER embedded in JWT claims. See Ch 14.
Redshift
Amazon Redshift — a cloud data warehouse. Used by Zeotap as a secondary analytics store for customer-facing reporting dashboards in Sisense, with data ingested via redshift-reporting-cron. See Ch 11.
Relabel Config
Prometheus configuration that transforms or filters metric labels during scraping. Used extensively in Zeotap's GCE service discovery to extract GCE labels (role, application, team, owner) as Prometheus labels. See Ch 16.
REST Assured
A Java DSL for testing REST APIs. Used by Zeotap's api-automation and backend-automation-framework test suites for E2E API testing of SmartPixel, ZAI, ZAT, and RBAC endpoints. See Ch 18 — Testing.

S

SBT (Scala Build Tool)
The primary build tool for Scala/Play Framework projects at Zeotap. Used in CircleCI pipelines to compile, test, and package JARs for services like audiences, profile-store, and gdpr-backend. See Ch 17.
ScalaTest
The most popular testing library for Scala. Used by Zeotap for unit and integration tests of Spark-based services like Profile Store, combined with spark-testing-base for SparkContext lifecycle management. See Ch 18.
Scylla
A high-performance, drop-in compatible replacement for Apache Cassandra, written in C++. Used by Zeotap's data lake for high-throughput reads/writes. Monitored via Prometheus exporter on port 9180. See Ch 16.
Secret Manager
Google Cloud Secret Manager — a service for securely storing API keys, OAuth tokens, passwords, and certificates. All Zeotap microservices retrieve credentials via Secret Manager at runtime, injected as environment variables via K8s secretKeyRef. See Ch 15.
Segment Estimator
A Go microservice (segmentestimator) that provides real-time audience size estimates using distributed bitmap aggregation across sharded nodes. Powers the Audience Builder's live count UI. See Ch 6.
Sisense
A business intelligence and analytics platform. Zeotap embeds Sisense dashboards in the Unity Angular frontend using JWT-authenticated iframe embedding, providing customer-facing analytics. See Ch 11.
SLO (Service Level Objective)
A measurable target for service reliability, e.g., "99.9% of API requests must return in < 500ms" or "error rate must be < 1%". Zeotap defines SLOs as Prometheus alert rules evaluated every 5 minutes. See Ch 16.
SmartPixel
Zeotap's client-side JavaScript tag for event collection on web properties. SmartPixel-3P extends this with server-side pixel firing via the Meta Conversions API, enabling cookieless conversion tracking. See Ch 8.
Symphony
Zeotap's internal workflow engine for Customer Journeys. Defines a JSON schema for multi-step marketing automation workflows with branching, delays, and destination actions. See Ch 7.

T

TCF (Transparency and Consent Framework)
See IAB TCF.
Terraform
An open-source Infrastructure as Code tool by HashiCorp. Zeotap uses Terraform for all GCP and AWS resource provisioning, organised into reusable modules in the gcp-terraform-modules repository. See Ch 15.
Terragrunt
A thin wrapper for Terraform that provides DRY (Don't Repeat Yourself) configuration reuse across environments (dev/qa/prod). Used to share project IDs, regions, and labels across all Zeotap Terraform modules without duplication. See Ch 15.
Time Travel
A Delta Lake feature enabling queries on historical snapshots of a table using VERSION AS OF or TIMESTAMP AS OF syntax. Used for audit, debugging, and point-in-time profile reconstruction. See Ch 5.

U

Unified Customer Profile (UCP)
A single, consolidated record for an individual that aggregates identity, behavioural events, and enrichment attributes from all sources. Stored in the MDM Delta table, keyed by zt_id. See Ch 5.
Unity
Zeotap's web application frontend — an Angular SPA with NgRx state management and lazy-loaded modules for all 7 product areas (ZAT, Insights, Studio, Taxonomy, Connect, Destinations, DPUI). See Ch 12 — Unity Dashboard.

V

ValidationProcessor
A Scala Spark processor in the Profile Store merge pipeline that validates incoming data quality (schema compliance, null checks, format validation) before records are merged into the MDM Delta table. See Ch 5.
Vertex AI
Google Cloud's unified ML platform for training, deploying, and managing ML models. Used by Zeotap for lookalike model training, propensity model training, and the Matching Engine for ANN search in Atlas AI. See Ch 10.
VPC (Virtual Private Cloud)
An isolated network segment within GCP. Zeotap's production infrastructure runs in a custom VPC (zeotap-vpc-prod) with private Google access enabled, ensuring all traffic stays on Google's backbone network. See Ch 15.

W

Workflow Creator Agent
The first sub-agent in Zeotap's journeys-ai SequentialAgent. Generates a Symphony Workflow JSON from a natural language description using Gemini 2.5 Flash and tool-calling against the client's catalog. See Ch 7.

X

XGBoost
An optimised gradient boosting library for supervised learning on tabular data. Used alongside LightGBM in Zeotap's propensity and churn prediction models. See Ch 10.

Z

ZAI (Zeotap AI)
Zeotap's AI enrichment service providing taxonomy classification and ML attribute generation via REST API. Exposes /v1/classify for IAB taxonomy and /v1/score for propensity prediction. See Ch 4 — GenAI & Zoe.
ZAT (Zeotap Audience Targeting)
The primary audience management product module within Unity. Provides the Audience Builder drag-and-drop UI, segment estimation, and audience export functionality. Product ID: 5. See Ch 12.
Zeotap MCP (Model Context Protocol)
A microservice exposing Zeotap's data assets to Claude-compatible AI assistants via the Anthropic Model Context Protocol. Runs in the ai GKE namespace (zeotap-mcp, 2 pods). See Ch 4.
Zoe
Zeotap's GenAI assistant embedded in the Unity dashboard. Powered by Claude (Anthropic) via the MCP protocol, enabling natural language queries over customer data, segment creation, and insights generation. See Ch 4 — GenAI & Zoe.
zt_id
Zeotap's canonical, internally assigned identifier for a unified customer profile. All identifiers (email SHA-256, IDFA, cookie IDs, etc.) resolve to a single zt_id via the ID Graph. Primary key in the MDM Delta table. See Ch 3 and Ch 5.
ZUILibrary
Zeotap's internal Angular component library and design system, shared across all product modules in Unity. Provides consistent UI components, typography, colour tokens, and layout primitives. See Ch 12.
