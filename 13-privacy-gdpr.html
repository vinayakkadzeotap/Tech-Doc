Privacy Architecture Overview

Zeotap is built with privacy-by-design: consent is captured at every data entry point, propagated through all pipelines, and enforced before any data is used for profiling, audience building, or activation. The privacy system spans four repositories:

  • gdpr-backend — Core GDPR API: opt-out management, deletion requests, audit logs (Play/Java + PostgreSQL)
  • consent-preference-centre-ui — Customer-facing consent management UI
  • zeotap-optin — Consent signal capture SDK (embedded in publisher pages)
  • flask-gdpr — Lightweight Python GDPR utility service for deletion orchestration
Consent Gate — Every data pipeline (ingestion, identity resolution, audience building, activation) checks the user's consent status before processing. If consent is absent or revoked, the UCID is excluded from all downstream processing.

GDPR Backend

The gdpr-backend is a Play Framework (Java) service that manages all GDPR-related data subject requests. It uses PostgreSQL for state and publishes deletion events to downstream services.

DpmIduJoinProcessor (Profile Store)

The Profile Store's deletion handler (Scala Spark) processes erasure events by joining deletion requests with IDU decisions and removing matching rows from the Delta table:

// profile-store/profile-merge/deletionHandler/DpmIduJoinProcessor.scala
class DpmIduJoinProcessor {
  def process(
    deletionRequests: Dataset[DeletionRequest],
    profileData: DataFrame,
    outputPath: String
  )(implicit spark: SparkSession): Unit = {

    val ucidsToDelete = deletionRequests
      .select("ucid")
      .distinct()

    // DELETE matching rows from Delta table
    DeltaTable.forPath(outputPath)
      .delete(col("ucid").isin(ucidsToDelete.collect().map(_.getString(0)): _*))

    // Vacuum to physically remove deleted data files
    // (after retention period)
    DeltaTable.forPath(outputPath).vacuum(0)
  }
}

IAB TCF v2.2 Integration

Zeotap is fully compliant with the IAB Transparency & Consent Framework (TCF) v2.2. All SDK-collected events carry the user's TCF consent string, and all downstream processing respects the declared purposes and vendor consents.
