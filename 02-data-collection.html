Data Collection & SDKs

Zeotap's collection layer spans web, mobile, and server-side environments. The analytics-sdk (JS), ZeotapCollect (iOS/Swift), and Android SDK provide unified event tracking, identity management, and consent enforcement across all client surfaces.

analytics-sdk v4.4.5
ZeotapCollect (Swift)

Overview

Zeotap provides four data collection mechanisms, each designed for a specific integration surface:

🌐
Collect SDK (Web)

JavaScript library for web event tracking. Tracks page views, custom events, user identities, and consent signals.

🍎
ZeotapCollect (iOS)

Native Swift package for iOS apps. Supports offline queuing, background flush, and IDFA/IDFV capture.

🤖
Android SDK

Kotlin/Java Android library with automatic screen tracking, lifecycle events, and GAID resolution.

🏷️
GTM Template

Google Tag Manager community template for no-code Zeotap integration without SDK deployment.

Web SDK — JavaScript

The Collect SDK is the primary web integration. It loads asynchronously and queues events until the SDK initializes.

Installation

html
<!-- Place before </head> -->
<script>
!function(e,t){
  var n=e.zeotap=e.zeotap||{};
  n._q=n._q||[];
  var s=["init","setUserId","track","page","setUserProperties"];
  s.forEach(function(e){
    n[e]=function(){n._q.push([e].concat(Array.prototype.slice.call(arguments)))}
  });
  var r=t.createElement("script");
  r.src="https://content.zeotap.com/sdk/latest/zeotap.js";
  r.async=true;
  t.getElementsByTagName("head")[0].appendChild(r);
}(window,document);

// Initialize with your Write Key
zeotap.init("YOUR_WRITE_KEY", {
  region: "EU",
  consentRequired: true
});
</script>

GTM Community Template

Search for "Zeotap" in the Google Tag Manager Template Gallery or import the template from zeocollect-gtm-template/ in the repository. No code deployment required.

1
Import Template

In GTM → Templates → Search Gallery → "Zeotap Collect". Or upload template.tpl from zeocollect-gtm-template/.

2
Configure Tag

Add your Write Key and select trigger (All Pages). Map GTM Data Layer variables to Zeotap event properties.

3
Publish

Preview, test with GTM Preview mode, then publish. The Zeotap SDK loads asynchronously on all matched triggers.

Core API Methods

javascript
// ① Identify a user
zeotap.setUserId("user_12345", {
  hashMethod: "SHA-256"   // automatic PII hashing
});

// ② Set user profile properties
zeotap.setUserProperties({
  email: "user@example.com",   // auto-hashed
  name: "Jane Doe",
  plan: "premium",
  age: 32
});

// ③ Track a page view
zeotap.page({
  title: "Product Detail Page",
  url: window.location.href,
  referrer: document.referrer
});

// ④ Track a custom event
zeotap.track("Product Added", {
  productId: "SKU-789",
  productName: "Wireless Headphones",
  category: "Electronics",
  price: 99.99,
  currency: "USD"
});

iOS SDK — Swift (ZeotapCollect)

ZeotapCollect is a native Swift package (Swift Package Manager) designed for iOS 13+ applications.

Installation via SPM

swift
// Package.swift — Add dependency
.package(
    url: "https://github.com/zeotap/ZeotapCollect",
    .upToNextMajor(from: "2.0.0")
)

Initialization & Usage

swift
import ZeotapCollect

// AppDelegate.swift
func application(_: UIApplication,
    didFinishLaunchingWithOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {

    // Initialize SDK
    Zeotap.initialize(
        writeKey: "YOUR_WRITE_KEY",
        configuration: ZeotapConfiguration(
            region: .EU,
            flushInterval: 30,          // seconds
            flushAt: 20,                // events
            trackApplicationLifecycleEvents: true
        )
    )
    return true
}

// Identify user
Zeotap.shared.identify(
    userId: "user_12345",
    traits: [
        "email": "user@example.com",
        "plan": "premium"
    ]
)

// Track event
Zeotap.shared.track(
    event: "Purchase Completed",
    properties: [
        "orderId": "ORD-9876",
        "revenue": 149.99,
        "currency": "USD"
    ]
)

Event Schema

All Zeotap events follow a common envelope schema. Every event carries a set of mandatory context fields alongside custom properties.

json
{
  // ── Envelope (auto-populated by SDK) ──
  "messageId":  "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "timestamp":  "2026-02-27T10:30:00.000Z",
  "type":       "track",           // "track" | "page" | "identify"
  "writeKey":  "wk_prod_xxxxxxxxxxxx",

  // ── Identity ──
  "userId":    "user_12345",
  "anonymousId": "anon_9f8e7d6c5b4a3210",

  // ── Event payload ──
  "event":     "Product Added",
  "properties": {
    "productId":   "SKU-789",
    "productName": "Wireless Headphones",
    "price":       99.99,
    "currency":    "USD"
  },

  // ── Context (auto-populated) ──
  "context": {
    "library":   { "name": "zeo-analytics", "version": "4.4.5" },
    "userAgent": "Mozilla/5.0 ...",
    "locale":    "en-US",
    "page":      {
      "url":      "https://shop.example.com/products/789",
      "title":    "Wireless Headphones — Shop",
      "referrer": "https://google.com"
    },
    "consent":   {
      "analytics": true,
      "marketing": false,
      "tcf":       "CO9n5aBO9n5..."     // TCF v2.2 consent string
    }
  }
}

Consent Management

The SDK integrates with CMP (Consent Management Platforms) via the IAB TCF v2.2 standard. Consent signals gate which data is collected and forwarded.

🔒 Consent Flow
flowchart LR A["🌐 Page Load"] --> B{"CMP\nPresent?"} B -->|Yes| C["Read TCF v2.2\nConsent String"] B -->|No| D["Implicit Consent\n(if configured)"] C --> E{"Consent\nGranted?"} D --> E E -->|Analytics ✓| F["Track Events"] E -->|Analytics ✗| G["Queue Events\n(pending consent)"] F --> H["Collect Endpoint"] G --> I{"Later consent\ngranted?"} I -->|Yes| F I -->|No| J["Discard Queue"] H --> K["🔗 ID Graph\nProcessing"] style F fill:#DCFCE7,stroke:#6EE7B7 style G fill:#FEF3C7,stroke:#FCD34D style J fill:#FEE2E2,stroke:#FCA5A5
javascript
// Manual consent update (e.g., after CMP interaction)
zeotap.updateConsent({
  analytics:  true,    // enable analytics tracking
  marketing:  true,    // enable marketing activation
  tcfString:  "CO9n5aBO..."  // raw IAB TCF v2.2 string
});

// Listen for CMP events automatically (OneTrust, Didomi, etc.)
zeotap.init("WRITE_KEY", {
  cmpIntegration: "auto",   // auto-detect CMP
  consentRequired: true
});
✅
GDPR Compliance Built-in

All Zeotap SDKs are designed with privacy by default. PII fields (email, phone) are SHA-256 hashed before transmission. Consent gates block data collection when not granted. See Chapter 13: Privacy & GDPR for server-side enforcement.
