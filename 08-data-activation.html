Overview

The Data Activation layer is the Activate pillar of Zeotap's CDP — it takes unified customer profiles and audience segments and delivers them to external marketing and advertising platforms. The core service, data-destination (Play Framework / Java), manages destination configurations, credentials, and the channel catalog.

Zeotap supports activation to:

  • Paid Media — Facebook, Google, The Trade Desk, Amazon DSP, Xandr, etc.
  • Email Platforms — Salesforce Marketing Cloud, Braze, HubSpot, Mailchimp
  • CRM Systems — Salesforce CRM, Microsoft Dynamics
  • Cloud Storage — AWS S3, GCS, Azure Blob, SFTP/SCP
  • Custom Webhooks — Generic HTTP endpoint push

Destination Types

Supported Secret / Credential Types

Each destination uses one of the following credential types, validated at creation time:

public enum SecretType {
    S3,      // accessKey + secretKey (required)
    SFTP,    // password OR pem_file (one required)
    SCP,     // pem_file (required)
    GCP,     // json_file (service account JSON)
    AZURE,   // key (connection string)
    OAUTH,   // oauthClientId + oauthClientSecret + isDynamic flag
    DEFAULT  // no credentials (public endpoints)
}
Type Use Case Credentials Required
S3 AWS S3 audience file delivery Access Key + Secret Key
SFTP Legacy system file drop Password or PEM certificate
SCP Secure copy to on-premise PEM certificate
GCP GCS bucket delivery Service Account JSON
AZURE Azure Blob Storage Connection string key
OAUTH Facebook, Google Ads, TTD Client ID + Secret + token

Integration Partners

Integration Partners are the set of supported platforms that Zeotap can activate to. Each partner has a channel catalogue mapping that defines what data types they accept (hashed emails, device IDs, cookie IDs) and in what format (CSV, API batch, streaming).

Secret Management

Destination credentials are never stored in plaintext. The service uses an encrypted secret vault with per-organisation isolation. OAuth tokens are stored and refreshed automatically:

// SecretUpsertRequestConstraint validates all credential upserts
// OauthController manages OAuth token lifecycle

// Create OAuth token
POST /channelSettings/api/v1/oauth/create
{
  "integration_partner_id": "facebook",
  "is_dynamic": false,
  "oauth_client_id": "12345678901234",
  "oauth_client_secret": "abcdef1234567890",
  "redirect_uri": "https://app.zeotap.com/oauth/callback"
}

// Exchange auth code for access token
POST /channelSettings/api/v1/oauth/token
{
  "integration_partner_id": "facebook",
  "code": "EAABC...",
  "org_id": 4521
}

For non-OAuth destinations, credentials are encrypted using AES-256 before storage and decrypted only at activation time within the connector process.

SmartPixel — Server-Side 3P Pixel Firing

The SmartPixel-3P module enables server-side firing of third-party tracking pixels (Facebook, Google, etc.) without relying on client-side JavaScript. This bypasses ad blockers and improves measurement accuracy.
