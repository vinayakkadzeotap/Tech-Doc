From Event to Activated Audience
A narrative walk-through of how the CDP processes a single user event end-to-end
Architecture Story
The lifecycle of one user action
1
User fires an event → Data Collection
A page-view, click, or custom event is captured by the JS SDK, iOS/Android native SDK, or server-side API. The payload is validated against the configured schema and compressed before being dispatched over HTTPS to the Collection endpoint.
2
Streaming layer buffers the event → Data Ingestion
The Collection service writes the event to an Apache Kafka topic (MSK). Kafka guarantees durability with at-least-once delivery. Upstream back-pressure is absorbed here, decoupling ingest rate from downstream processing throughput.
3
Flink validates and enriches → Data Ingestion
Apache Flink 1.18 streaming jobs consume from Kafka with event-time windowing. Payloads are schema-validated (Avro / JSON Schema), geo-enriched, and consent-gated before being forwarded downstream. Invalid events are sent to a DLQ for alerting.
4
Identity Graph stitches the user → Identity Resolution
id-graph-lib-v2 (Go) performs deterministic matching on known identifiers (email, phone) then probabilistic resolution via device fingerprinting. The resolution result is a Unified Customer ID (UCID) that links all known identifiers in BigTable, cached in Redis for sub-ms hot path reads.
5
Profile is updated → Profile Store
Resolved events are written to the unified Profile Store. Scylla DB provides the hot-path read/write (p99 <8ms). Event attributes are merged into the customer profile ledger, and BigQuery receives an async copy for analytical queries via the twin-sync pattern.
6
Segmentation engine re-evaluates membership → Segmentation Engine
A profile-change event triggers the Rust-native rule evaluation engine. Rules covering 800+ condition types are evaluated against the updated profile. Segment membership changes are emitted as changelog events, enabling real-time audience updates within seconds.
7
Journey checks for triggers → Journeys & Orchestration
The Airflow-backed journey executor evaluates trigger conditions against the segment changelog. If the user just entered a qualifying segment or fired a trigger event, the next journey step is enqueued — an email, push, wait node, or A/B branch decision.
8
Activation pushes to destination → Activation Layer
connector-hub (Scala) picks up the export job, maps identity fields to the destination's expected format (GAID, email hash, IDFA, etc.), and delivers to one of 200+ ad platforms, CRMs, or cloud warehouses. OAuth refresh, rate-limit management, and retry queues are handled automatically.

Common Failure Points
Known failure modes per architecture layer — use Debug Mode to highlight these
⚡
Data Ingestion
  • Schema mismatch — event field types don't match registry
  • Ingestion delay — Kafka consumer lag spike during traffic bursts
  • Invalid payload — missing required fields (user_id, event_type)
  • DLQ overflow — too many malformed events reaching dead-letter queue
  • Batch file parsing failure — CSV/NDJSON encoding errors
🔗
Identity Resolution
  • Identifier collision — two users merged under one UCID
  • Missing identifiers — anonymous events not linked to known user
  • BigTable hotspot — skewed key distribution slows resolution
  • Redis cache miss — cold-start resolution latency spike
  • Graph inconsistency — stale edges after DSR erasure
🎯
Segmentation
  • Stale segment computation — batch job runs behind schedule
  • Attribute type mismatch — string vs integer in rule conditions
  • Rule evaluation timeout — overly complex multi-condition rules
  • Flink checkpoint failure — streaming segment update delayed
  • BigQuery quota exceeded — long-running SQL segment query blocked
🚀
Activation
  • API rejection — destination returns 4xx on malformed payload
  • Identifier mapping errors — GAID / email hash not found on profile
  • OAuth token expiry — connector refresh fails silently
  • Rate limit breach — destination throttles bulk export
  • Segment export job stalls — Kafka lag in connector-hub topic
