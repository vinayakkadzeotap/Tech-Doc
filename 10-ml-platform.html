Platform Overview

The Zeotap ML Platform enables the CDP to go beyond raw data storage and segment-building — it generates predictive intelligence at the individual profile level. This includes:

  • Propensity Models — Purchase propensity, churn risk, email open likelihood
  • LTV Segmentation — Customer lifetime value prediction (high/mid/low/at-risk)
  • Lookalike Audiences — Finding users who resemble a seed audience (Atlas AI)
  • AI Attributes — Model-derived attributes attached to every profile
  • NLP Discovery — Extracting audience attributes from unstructured text (product descriptions, reviews)

MLOps Pipeline

The mlops repository contains the orchestration layer for training, evaluating, and deploying ML models at scale on GCP. It uses Apache Beam (running on Dataflow) for distributed feature engineering and Vertex AI Pipelines for training orchestration.

Pipeline Stages

1
Feature Engineering (Beam / Dataflow) — Reads raw profile data from Delta Lake, computes rolling aggregates (7d, 30d, 90d page views, purchases, sessions), and writes feature vectors to BigQuery feature store.
2
Model Training (Vertex AI) — Triggered on schedule (weekly) or on-demand. Trains XGBoost/LightGBM propensity models and BG/NBD LTV models using BigQuery training datasets.
3
Evaluation & Champion/Challenger — New model versions are evaluated against held-out test sets (AUC, precision@k, RMSE). A champion/challenger framework determines which version promotes to production.
4
Batch Scoring — The production model scores all UCID profiles via a Beam job, writing predicted scores (purchase_propensity, churn_probability, ltv_segment) back to the Profile Store.

Atlas AI — Lookalike Audience Engine

atlas-ai is Zeotap's proprietary lookalike modelling system. Given a seed audience (e.g., "our top 5,000 buyers"), Atlas AI finds similar users in the broader profile database using embedding-based cosine similarity.

# AI Attributes Beam pipeline (simplified)
class ScoreProfilesFn(beam.DoFn):
    def setup(self):
        self.model = load_model("gs://ml-models/propensity/v7/model.json")

    def process(self, profile):
        features = extract_features(profile)
        score = self.model.predict_proba([features])[0][1]
        yield {
            "ucid": profile["ucid"],
            "purchase_propensity": round(score, 4),
            "scored_at": datetime.utcnow().isoformat()
        }

# Run scoring pipeline on Dataflow
with beam.Pipeline(options=DataflowPipelineOptions()) as p:
    (p
     | "ReadProfiles" >> ReadFromBigQuery(query=PROFILE_QUERY)
     | "ScoreProfiles" >> beam.ParDo(ScoreProfilesFn())
     | "WriteScores"  >> WriteToBigQuery(
         table="project.dataset.ai_attributes",
         write_disposition=BigQueryDisposition.WRITE_TRUNCATE)
    )

NLP Discovery (discover-nlp)

The discover-nlp service uses transformer-based NLP to extract structured audience attributes from unstructured text — product catalogues, editorial content, user reviews — enabling contextual targeting without explicit user labelling.

Use case: A publisher ingests article content → NLP service extracts topics (e.g., "electric vehicles", "luxury travel") → these are mapped to IAB content categories → users reading these articles receive topic affinity attributes on their profile.
# discover-nlp API
POST /nlp/classify
